# LESSONS

| Date       | What went wrong                 | Rule to apply                          |
|------------|---------------------------------|----------------------------------------|
| 2026-06-02 | `useId()` appelé dans des Server Components `async` pour générer des keys, alors que les docs Sanity ont déjà `_id` | Server Component `async` = pas de hooks React. Utiliser les identifiants stables des données (`_id`) comme `key`. |
| 2026-06-02 | `urlFor('')` lève « Unable to resolve image URL » si l'image Sanity manque → crash SSR ; le `\|\| fallback` après `.url()` ne protège pas (erreur levée avant) | Garder `urlFor()` derrière un garde d'existence : `image?.asset ? urlFor(image)... : fallbackStatique`. Ne jamais passer une source vide. |
| 2026-06-02 | Studio Sanity embarqué dans Next (port 3000) → `CorsOriginError` : seul `localhost:3333` (port du Studio standalone) était whitelisté | Studio embarqué = ajouter le port réel de l'app aux CORS : `sanity cors add http://localhost:3000 --credentials` (+ l'URL de prod au déploiement) |
| 2026-06-02 | Migration majeure Next 15→16 : cascade de peers (next-sanity 11→13 → sanity 4→5 → @sanity/client direct), découverte étape par étape via les warnings pnpm | Après un bump majeur, lire les peer-deps warnings et migrer toute la cascade ; vérifier next-sanity / sanity / @sanity/client ensemble |
| 2026-06-02 | next-sanity 13 : `sanityFetch` renvoyait `data: unknown` car l'augmentation `declare module '@sanity/client'` échouait (TS2664 module introuvable) | Installer `@sanity/client` en dépendance DIRECTE pour que l'augmentation de types générée par typegen fonctionne |
| 2026-06-02 | `sanity schema extract` cassé en sanity 5.28 / @sanity/cli 6.7 (`getWorkspace is not a function`, bug upstream ESM) → `pnpm typegen` complet échoue | Contournement : `typegen` = `sanity typegen generate` seul (lit extract.json existant) ; `schema extract` isolé dans `schema:extract`, à relancer quand le schéma change / le bug upstream est corrigé |
| 2026-06-02 | Next 16 : `next lint` supprimé + Turbopack par défaut | Lint = ESLint CLI en flat config native (`eslint-config-next/core-web-vitals` + `/typescript`, sans FlatCompat) ; SVGR via `turbopack.rules`, plus de `webpack()` |
| 2026-06-02 | Theme toggle next-themes : le garde `mounted` via `useEffect(() => setMounted(true))` déclenche la règle ESLint `react-hooks/set-state-in-effect` | Piloter l'affichage des icônes du toggle en CSS (`hidden dark:block` / `dark:hidden`) plutôt qu'en JS — pas de useState/useEffect, pas de mismatch d'hydratation |
| 2026-06-03 | Bug `schema extract` (`getWorkspace is not a function`) **résolu** en montant `sanity` 5.28 → **5.30.0** ; bonus : l'ancien `extract.json` était périmé (15 types → 17 réels) | Avant un contournement durable sur une lib, re-tester après un bump mineur. Flux types : `pnpm schema:extract` (régénère `extract.json`) **puis** `pnpm typegen`, à relancer à chaque changement de structure du schéma |
| 2026-06-03 | Refonte design : next-themes en `attribute="class"` ne matchait pas le CSS du handoff en `[data-theme]` | Aligner la stratégie de thème sur le CSS livré : `attribute="data-theme"` ; brancher les fonts `next/font` sur les tokens via `--font-*` (zéro `@import` Google) |
| 2026-06-03 | Modélisation i18n : tentation de créer un type/doc par langue ou de coder les langues en dur | Textes traduisibles = `internationalizedArrayString/Text` (plugin) ; langues + catégories = **data** (docs `language`/`skillCategory`). Un wrapper fin `defineField({type:'internationalizedArrayString'})` compile sans cast. Singletons = `structure.ts` (documentId fixe) + `document.newDocumentOptions` pour bloquer les doublons. |
| 2026-06-04 | Routing `[locale]` avec zones localisées **et** non localisées (Studio) → besoin de `<html lang>` différents | **Multiple root layouts** : supprimer `app/layout.tsx`, mettre chaque zone dans un groupe (`(site)/[locale]/layout.tsx`, `(utility)/layout.tsx`) qui rend son propre `<html><body>`. Sous-groupe `(chrome)` pour la nav partagée. `/` → middleware/proxy (pas de `app/page.tsx` possible). |
| 2026-06-04 | `next build` : « The "middleware" file convention is deprecated. Use "proxy" instead. » (Next 16.2) | Renommer `middleware.ts` → `proxy.ts` et la fonction `export function middleware` → `export function proxy` ; `export const config = { matcher }` inchangé. |
| 2026-06-04 | Déplacer des fichiers dont le chemin contient `()[]` (route groups, segments dynamiques) : `git mv` glob le `[]` (pathspec), `Move-Item` sans `-LiteralPath` aussi | Utiliser **`[System.IO.File]::Move(src,dst)`** (+ `Directory.CreateDirectory`) en chemins absolus littéraux, puis `git add -A` (git détecte le rename). Les outils Read/Write/Edit gèrent les `()[]` nativement. |
| 2026-06-04 | Locales pilotées par Sanity : si `generateStaticParams` lit un dataset vide, le site 404 partout avant le bootstrap | `getLanguages()` (React `cache`, `try/catch`) retombe sur un fallback `['en']` si Sanity est vide/injoignable. Locales = data, mais dégradation gracieuse — jamais de 404 faute de docs. |
| 2026-06-04 | `next dev` (Turbopack) boucle en `FATAL ... Failed to write app endpoint /(frontend)/page` / `Cell ... AppPageLoaderTree no longer exists` après le refactor de routes — alors que `pnpm build` passe | Cache incrémental persistant Turbopack (`.next`) périmé, référence encore une route déplacée/supprimée. Ce n'est PAS un bug de code (prod intacte). Fix : couper le dev, **`Remove-Item .next -Recurse -Force`**, relancer. Réflexe : purger `.next` après tout déplacement/suppression de fichiers de routes ou de route groups. |
| 2026-06-05 | i18n : changer de langue ne changeait jamais le contenu. `pickLocale` matchait `item._key`, mais `sanity-plugin-internationalized-array` stocke la locale dans le champ **`language`** (`_key` = hash aléatoire) → toujours le 1er élément renvoyé | Résoudre les champs i18n via `e.language === locale` (fallback `_key`). **Toujours vérifier la forme RÉELLE des données** (`*[_type=='x'][0]{field}` sur l'API) avant d'écrire un résolveur — ne pas supposer la convention de clé du plugin. |
| 2026-06-05 | Sélecteur de langue invisible/clippé : mon dropdown réutilisait la classe `.lang` du design, qui a `overflow:hidden` (c'est un toggle segmenté, pas un menu) → menu coupé | Vérifier l'`overflow` des conteneurs ancêtres avant de poser un popover ; utiliser une **classe dédiée** sans `overflow:hidden`. Pour scaler à beaucoup de langues : dropdown avec menu `max-height`+scroll, pas un toggle qui s'étale. |
| 2026-06-05 | Erreur console `Encountered a script tag while rendering React component` pointant ThemeProvider, avec `chrome-extension://…/inpage.js` dans la stack | Origine = **extension navigateur** (wallet crypto), pas le code : confirmé par l'absence d'erreur console dans un aperçu propre. Tester en navigation privée / sans extensions avant de chasser ce type d'erreur. |
