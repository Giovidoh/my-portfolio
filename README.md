# Portfolio

Personal portfolio built with **Next.js 16** (App Router) and **Sanity v5** as a headless CMS, with the Studio embedded at `/studio`.

## Stack

- [Next.js 16](https://nextjs.org) (App Router, Turbopack) + React 19.2
- [Sanity v5](https://www.sanity.io) (`next-sanity` 13) — content + embedded Studio
- Tailwind CSS v4
- TypeScript, ESLint (flat config)

## Getting started

### 1. Prerequisites

- Node.js **>= 20.9**
- [pnpm](https://pnpm.io)

### 2. Environment

Copy `.env.example` to `.env.local` and fill in your Sanity project values:

```bash
cp .env.example .env.local
```

```
NEXT_PUBLIC_SANITY_PROJECT_ID=your_project_id
NEXT_PUBLIC_SANITY_DATASET=production
```

Create a project at [sanity.io/manage](https://www.sanity.io/manage). Add `http://localhost:3000`
to the project's **CORS origins** (with credentials) so the embedded Studio can authenticate:

```bash
pnpm exec sanity cors add http://localhost:3000 --credentials
```

### 3. Install & run

```bash
pnpm install
pnpm dev
```

- App: http://localhost:3000
- Studio: http://localhost:3000/studio

Add your content in the Studio: a **Profile**, some **Skills**, **Projects**, and an optional
**Sections Config** (custom section headings).

## Scripts

| Script | Description |
| --- | --- |
| `pnpm dev` | Start the dev server (runs `typegen` first) |
| `pnpm build` | Production build |
| `pnpm start` | Run the production build |
| `pnpm lint` | ESLint |
| `pnpm typegen` | Generate TypeScript types from the extracted schema + GROQ queries |
| `pnpm schema:extract` | Re-extract the Sanity schema to `sanity/extract.json` |

## Type generation

Types in `sanity/types.ts` are generated by Sanity TypeGen. `pnpm typegen` runs
`sanity typegen generate`, which reads `sanity/extract.json` and your `defineQuery` queries.

> **Note:** `pnpm schema:extract` (regenerating `sanity/extract.json` from the Studio config)
> currently fails on Sanity 5.28 / `@sanity/cli` 6.7 with `getWorkspace is not a function` — an
> upstream toolchain bug. The committed `sanity/extract.json` matches the current schema, so
> `pnpm typegen` works as-is. If you change the schema structure, `schema:extract` must succeed
> first (track the upstream fix).

## Deployment

Add your production URL to the Sanity CORS origins before deploying:

```bash
pnpm exec sanity cors add https://your-domain.com --credentials
```
